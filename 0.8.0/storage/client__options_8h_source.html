26 /**
27  * Describes the configuration for a `storage::Client` object.
28  *
29  * By default, several environment variables are read to configure the client:
30  *
31  * - `CLOUD_STORAGE_TESTBENCH_ENDPOINT`: if set, use this http endpoint to
32  * make all http requests instead of the production GCS service. Also,
33  * if set, the `CreateDefaultClientOptions()` function will use an
34  * `AnonymousCredentials` object instead of loading Application Default
35  * %Credentials.
36  * - `CLOUD_STORAGE_ENABLE_CLOG`: if set, enable std::clog as a backend for
37  * `google::cloud::LogSink`.
38  * - `CLOUD_STORAGE_ENABLE_TRACING`: if set, this is the list of components that
39  * will have logging enabled, the component this is:
40  * - `http`: trace all http request / responses.
41  */
