Remove stale buckets created for examples.

The examples and integration tests create buckets in the production environment. While these programs are supposed to clean after themselves, they might crash or otherwise fail to delete any buckets they create. These buckets can accumulate and cause future tests to fail (see #4905). To prevent these problems we delete any bucket that match the pattern of these randomly created buckets, as long as the bucket was created more than 48 hours ago.

Parameters
clientused to make calls to GCS.
prefixonly delete buckets that start with this string followed by a date (in YYYY-mm-dd format), and then an underscore character (_).
created_time_limitonly delete buckets created before this timestamp.
