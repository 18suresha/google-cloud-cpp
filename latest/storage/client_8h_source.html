41 /**
42  * The Google Cloud Storage (GCS) Client.
43  *
44  * This is the main class to interact with GCS. It provides member functions to
45  * invoke all the APIs in the service.
46  *
47  * @par Performance
48  * Creating an object of this type is a relatively low-cost operation.
49  * Connections to the service are created on demand. Copy-assignment and
50  * copy-construction are also relatively low-cost operations, they should be
51  * comparable to copying a few shared pointers. The first request (or any
52  * request that requires a new connection) incurs the cost of creating the
53  * connection and authenticating with the service. Note that the library may
54  * need to perform other bookeeping operations that may impact performance.
55  * For example, access tokens need to be refreshed from time to time, and this
56  * may impact the performance of some operations.
57  *
58  * @par Thread-safety
59  * Instances of this class created via copy-construction or copy-assignment
60  * share the underlying pool of connections. Access to these copies via multiple
61  * threads is guaranteed to work. Two threads operating on the same instance of
62  * this class is not guaranteed to work.
63  *
64  * @par Credentials
65  * The default approach for creating a Client uses Google Application Default
66  * %Credentials (ADCs). Because finding or loading ADCs can fail, the returned
67  * `StatusOr<Client>` from `CreateDefaultClient()` should be verified before
68  * using it. However, explicitly passing `Credentials` when creating a Client
69  * does not have the same potential to fail, so the resulting `Client` is not
70  * wrapped in a `StatusOr`. If you wish to use `AnonymousCredentials` or to
71  * supply a specific `Credentials` type, you can use the functions declared in
72  * google_credentials.h:
73  * @code
74  * namespace gcs = google::cloud::storage;
75  *
76  * // Implicitly use ADCs:
77  * StatusOr<gcs::Client> client = gcs::Client::CreateDefaultClient();
78  * if (!client) {
79  * // Handle failure and return.
80  * }
81  *
82  * // Or explicitly use ADCs:
83  * auto creds = gcs::oauth2::GoogleDefaultCredentials();
84  * if (!creds) {
85  * // Handle failure and return.
86  * }
87  * // Status was OK, so create a Client with the given Credentials.
88  * gcs::Client client(gcs::ClientOptions(*creds));
89  *
90  * // Use service account credentials from a JSON keyfile:
91  * std::string path = "/path/to/keyfile.json";
92  * auto creds =
93  * gcs::oauth2::CreateServiceAccountCredentialsFromJsonFilePath(path);
94  * if (!creds) {
95  * // Handle failure and return.
96  * }
97  * gcs::Client client(gcs::ClientOptions(*creds));
98  *
99  * // Use Compute Engine credentials for the instance's default service account.
100  * gcs::Client client(
101  * gcs::ClientOptions(gcs::oauth2::CreateComputeEngineCredentials()));
102  *
103  * // Use no credentials:
104  * gcs::Client client(
105  * gcs::ClientOptions(gcs::oauth2::CreateAnonymousCredentials()));
106  * @endcode
107  *
108  * @par Error Handling
109  * This class uses `StatusOr<T>` to report errors. When an operation fails to
110  * perform its work the returned `StatusOr<T>` contains the error details. If
111  * the `ok()` member function in the `StatusOr<T>` returns `true` then it
112  * contains the expected result. Please consult the
113  * [`StatusOr<T>` documentation](#google::cloud::v0::StatusOr) for more details.
114  *
115  * @code
116  * namespace gcs = google::cloud::storage;
117  * gcs::Client client = ...;
118  * google::cloud::StatusOr<gcs::BucketMetadata> bucket_metadata =
119  * client.GetBucketMetadata("my-bucket");
120  *
121  * if (!bucket_metadata) {
122  * std::cerr << "Error getting metadata for my-bucket: "
123  * << bucket_metadata.status() << "\n";
124  * return;
125  * }
126  *
127  * // Use bucket_metadata as a smart pointer here, e.g.:
128  * std::cout << "The generation for " << bucket_metadata->name() " is "
129  * << bucket_metadata->generation() << "\n";
130  * @endcode
131  *
132  * In addition, the @ref index "main page" contains examples using `StatusOr<T>`
133  * to handle errors.
134  *
135  * @par Retry, Backoff, and Idempotency Policies
136  *
137  * The library automatically retries requests that fail with transient errors,
138  * and follows the
139  * [recommended
140  * practice](https://cloud.google.com/storage/docs/exponential-backoff) to
141  * backoff between retries.
142  *
143  * The default policies are to continue retrying for up to 15 minutes, and to
144  * use truncated (at 5 minutes) exponential backoff, doubling the maximum
145  * backoff period between retries. Likewise, the idempotency policy is
146  * configured to retry all operations.
147  *
148  * The application can override these policies when constructing objects of this
149  * class. The documentation for the constructors show examples of this in
150  * action.
151  *
152  * @see https://cloud.google.com/storage/ for an overview of GCS.
153  *
154  * @see https://cloud.google.com/storage/docs/key-terms for an introduction of
155  * the key terms used in GCS.
156  *
157  * @see https://cloud.google.com/storage/docs/json_api/ for an overview of the
158  * underlying API.
159  *
160  * @see https://cloud.google.com/docs/authentication/production for details
161  * about Application Default %Credentials.
162  *
163  * @see #google::cloud::v0::StatusOr.
164  *
165  * @see `LimitedTimeRetryPolicy` and `LimitedErrorCountRetryPolicy` for
166  * alternative retry policies.
167  *
168  * @see `ExponentialBackoffPolicy` to configure different parameters for the
169  * exponential backoff policy.
170  *
171  * @see `AlwaysRetryIdempotencyPolicy` and `StrictIdempotencyPolicy` for
172  * alternative idempotency policies.
173  */
