64 /***
65  * Set the maximum connection pool size.
66  *
67  * The C++ client library uses this value to limit the growth of the
68  * connection pool. Once an operation (a RPC or a download) completes the
69  * connection used for that operation is returned to the pool. If the pool is
70  * full the connection is immediately released. If the pool has room the
71  * connection is cached for the next RPC or download.
72  *
73  * @note The behavior of this pool may change in the future, depending on the
74  * low-level implementation details of the library.
75  *
76  * @note The library does not create connections proactively, setting a high
77  * value may result in very few connections if your application does not need
78  * them.
79  *
80  * @warning The library may create more connections than this option configures,
81  * for example if your application requests many simultaneous downloads.
82  */
