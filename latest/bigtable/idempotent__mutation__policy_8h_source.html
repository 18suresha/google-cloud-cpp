64 /**
65  * Implements a policy that retries all mutations.
66  *
67  * Notice that this will may result in non-idempotent mutations being resent
68  * to the server. Re-trying a SetCell() mutation where the server selects the
69  * timestamp can result in multiple copies of the data stored with different
70  * timestamps. Only use this policy if your application is prepared to handle
71  * such problems, for example, by only querying the last value and setting
72  * garbage collection policies to delete the old values.
73  */
74 class AlwaysRetryMutationPolicy : public IdempotentMutationPolicy {
75  public:
76  AlwaysRetryMutationPolicy() {}
